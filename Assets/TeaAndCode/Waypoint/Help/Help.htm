Basics

1.     Introduction

This tool is an all-in-one package for adding waypoints (a.k.a. objective markers or POIs) to your game. It has a built-in editor window to suit your needs and renders using Unity’s GUITextures.

 

2.     Setup

Once downloaded, navigate to the Waypoint Editor in the Tools dropdown menu in the Unity Toolbar. Select the Waypoint Editor to open up a new window with a description and a ‘Create Waypoint System’ button. Click this button to begin using waypoints. This window is where you can do the majority of the customization of waypoints. I recommend docking the window as a tab on the right side of the editor.

 

3.     Interface

·        Player

o   Set the game object in your scene that determines the position of the player’s character. It is used for distance calculations. If none is specified, the main camera is used instead for distance calculations.

·        Camera

o   Set the Camera in your scene that determines the render target for the waypoints. Defaults to Camera.main when played if unspecified.

·        Offscreen Style

o   Compass – place the waypoint around a circle when off screen

o   Clamp – place the waypoint on the edge of the screen when off screen

·        Safe Margin

o   Determines how close to the edge the waypoints will draw before it considers the object off-screen

·        Compass Size

o   Determines the size of the compass circle by height ratio [1 is the height of the screen]

·        Pulse period

o   Time in seconds for a pulse cycle when a waypoint is pulse mode

·        Falloff

o   Toggles waypoints fading out by distance

·        Falloff Max

o   This field is used for clamping the max range of Falloff Range.

·        Falloff Range

o   Sliders determine when the falloff alpha curve begins and ends. The key is to read the Min: and Max: numbers in the infobar just below it.

·        Falloff Alpha

o   This curve describes how falloff works. The value at 0 in the curve is the default alpha of waypoints on-screen. The value at 1 determines the final alpha of waypoints when they reach the max distance away from the player.

·        New

o   Creates a new waypoint at 0, 0, 0

·        New at selection

o   Creates a new waypoint as a child of the current selection

·        Clone

o   Clones the current selected waypoint

·        Delete

o   Deletes the current selected waypoint

·        Waypoint scrollview

o   Single-click selects the waypoint for editing

o   Double-click selects the waypoint in the scene

·        Name

o   Name of the waypoint

·        On Screen Widget

o   Prefab that determines the look and behavior of the waypoint when on-screen

·        Off Screen Widget

o   Prefab that determines the look and behavior of the waypoint when off-screen

·        Icon

o   Texture that is rendered onto the waypoint

·        Icon Tint

o   Tint the color of the icon. White means no tint.

·        Label

o   Text that displays on the waypoint

·        Font Color

o   Color of the text displayed on the waypoint

·        Display Style

o   Used to control visibility and pulsing of the waypoint

 

4.     Organization

It is imperative that the relative path stays at \Assets\TeaAndCode\Waypoint. If you want to change the tool location, edit the public static variable WaypointPath in Scripts\WaypointSystem.cs to reflect the change.

 

Scripts

Directory: \Waypoint\Scripts

This is where all of the C# scripts reside.

1.     WaypointEditor.cs: This script provides a user-friendly interface that allows you to create, modify, and manage instances of the following scripts.

2.     WaypointSystem.cs: This is the script that determines the high-level behaviors of the waypoint system. As it is a singleton, you should only ever have one gameobject with this component attached.

3.     Waypoint.cs: This is the script that describes the location and behavior of the waypoint. This script creates a widget on called WaypointWidget on start of the game. Modifying the properties of Waypoint.cs component will change the WaypointWidget in real-time.

4.     WaypointWidget.cs: This class is the abstract base class that actually positions and draws the visuals on screen. It receives the information it needs from Waypoint.cs.

5.     OnScreenWidget.cs: This class inherits from WaypointWidget and positions and draws the on-screen version of the waypoint.

6.     OffScreenWidget.cs: This class inherits from WaypointWidget and positions and draws the off-screen version of the waypoint.

7.     RotatableGUITexture.cs: This class is a rotatable GUITexture used for drawing the off-screen arrows.

 

Widgets

Directory: \Waypoint\Widgets

This is where the prefabs for the WaypointWidgets live. There is a dropdown box under Waypoint in the Waypoint Editor window that allows you to select these prefabs for use. One of the tutorials show how you can create custom prefabs of your own if the included ones do not fit your needs.

 

Icons

Directory: \Waypoint\Icons

Here are some icons to get you started with the Waypoint system. If you want to add your own icons, this directory is a good place to drop them.

 

Fonts

Directory: \Waypoint\Fonts

Included is one default font for your waypoint system. Just drop a font file in this folder to add more fonts.

 

Help

Directory: \Waypoint\Help

This is where the help documentation lives. You should never have to modify the contents of this older.

 

Gizmos

Directory: \Waypoint\Help

This directory contains the gizmo icon of the Waypoint object. You should never have to modify the contents of this older.

 

Resources

Directory: \Waypoint\Resources

This directory contains the Waypoint logo. You should never have to modify the contents of this older.

 

Tutorials

1.     My first waypoint

1.     If you have not done so, click Create Waypoint System

2.     Click the New button under the Waypoints foldout. A new waypoint should appear

3.     Move the waypoint in the scene to where you want in the world

4.     Go back to the Waypoint Editor

5.     Change the Icon to any of the icons in ‘Assets\TeaAndCode\Waypoints\Icons’

6.     Change the On Screen Widget from None to Simple

7.     Change the Off Screen Widget from None to Simple

8.     Change the Label if you desire

9.     All set. Hit Play to check it out in game

 

HELP! I can’t see my waypoint in-game!

·        Make sure the Icon field has a texture

·        Make sure the IconTint color has an alpha of greater than zero

·        Make sure the On Screen Widget is not set to None

·        Make sure the Off Screen Widget is not set to None

·        Try disabling falloff by unchecking the Falloff checkbox (the waypoint may be invisible because it is beyond the falloff max distance)

 

2.     Enemy waypoints

1.     Select the enemy in the scene

2.     Click the New at Selection in the Waypoint Editor

3.     Customize the waypoint in the Waypoint Editor

4.     If the enemy is a prefab, apply the prefab in the Inspector

 

And you’re done! The waypoint should automatically move along with the enemy and disappear when the enemy is destroyed.

 

3.     Sequential checkpoints

This tutorial showcases a sample script that interacts with the Waypoints dynamically and embedding a Waypoint in a prefab.

 

1.     Navigate to TeaAndCode\Waypoint\Sample\ in the Project tab

2.     Drag and drop the Checkpoint prefab into the scene where you want the first checkpoint

3.     Name the Checkpoint and its child Waypoint to something appropriate like Checkpoint_01 and CP_Waypoint_01

a.     Naming these appropriately will help you distinguish them on the Waypoint Editor later for editing

4.     Check the Start Enabled checkbox

5.     Customize the Checkpoint Waypoint (named CP_Waypoint_01 if you renamed it) in the Waypoint Editor if you desire. It should already have some default setup.

6.     Duplicate the Checkpoint in the scene for  the next checkpoint

7.     Rename the new Checkpoint appropriately (Checkpoint_02 and CP_Waypoint_02)

8.     Move Checkpoint_02 to the next location

9.     Select Checkpoint_01 and assign Checkpoint_02 as the Next Checkpoint

10.You now have a sequential checkpoint setup. You can set up as many checkpoints as you need.

 

4.     Create a new widget

1.     Find the Assets\TeaAndCode\Waypoint\Widgets\ folder in the Project tab

2.     Duplicate any of the prefabs

a.     Pick the one most similar to your needs

3.     Rename the duplicate to YourPrefabOnScreen if it has the OnScreenWidget component or YourPrefabOffScreen if it has the OffScreenWidget component

a.     Waypoint checks for the OnScreen and OffScreen at the end to determine what kind of widget it is

4.     Drag & drop the prefab into the scene.

5.     Switch to the Game tab from the Scene tab. You should see it appear in middle of the viewport

6.     Customize the prefab by editing the version in the scene through the Inspector

7.     Apply the prefab. It will now appear in the appropriate Widget dropdown in the Waypoint Editor

 

5.     Make my script talk to waypoints

When interfacing with HUD Waypoint, the instances of WaypointSystem and Waypoint are what you are concerned with. With the exception of the widget prefabs, you can change the behaviors of the waypoints by changing the public properties in real-time. These properties can be found inside the scripts’ Properties regions. I recommend having a serialized reference to whichever Waypoint or WaypointSystem you need in the script that needs to change them. For an example of this paradigm, take a look at TeaAndCode\Waypoint\Scripts\Sample\Checkpoint.cs.
